# 界面说明

## 布局结构

界面采用 **Todo List 卡片形式**，以项目为单位展示。

### 卡片布局

每个项目一行卡片，包含：

```
┌─────────────────────────────────────────────────────────────────┐
│  [图标]  项目名称                    [按钮1] [按钮2] [按钮3]     │
│          项目描述                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 项目列表

#### 1. spms-web (前端)
```
┌─────────────────────────────────────────────────────────────────┐
│  🎨  spms-web (前端)          [🚀 云效执行] [☸️ K8s发布] [⚡ 全部] │
│      前端项目 - 云效构建与测试环境部署                            │
└─────────────────────────────────────────────────────────────────┘
```

#### 2. spms-server (后端)
```
┌─────────────────────────────────────────────────────────────────┐
│  ⚙️  spms-server (后端)       [🚀 云效执行] [☸️ K8s发布] [⚡ 全部] │
│      后端项目 - 云效构建与测试环境部署                            │
└─────────────────────────────────────────────────────────────────┘
```

## 按钮功能

### 🚀 云效执行
- **功能**: 仅触发云效构建
- **流程**: 触发构建 → 获取版本号
- **适用场景**: 只想构建，不部署

### ☸️ K8s发布
- **功能**: 使用最近构建版本更新 K8s
- **流程**: 获取最近构建版本号 → 更新测试环境
- **适用场景**: 已有构建版本，直接部署

### ⚡ 全部
- **功能**: 完整部署流程
- **流程**: 触发构建 → 获取版本号 → 更新测试环境
- **适用场景**: 一键完成构建和部署

## 卡片状态

### 默认状态
- 白色背景
- 灰色边框

### 运行中
- 浅橙色背景
- 左侧橙色边框（4px）

### 成功
- 浅绿色背景
- 左侧绿色边框（4px）

### 失败
- 浅红色背景
- 左侧红色边框（4px）

## 日志区域

位于卡片列表下方，显示实时执行日志：
- 黑色背景
- 彩色日志级别标签（INFO/SUCCESS/WARNING/ERROR）
- 自动滚动到底部

## 通知提示

任务完成后，顶部中央显示通知：
- 成功：紫色渐变背景 + ✅ 图标
- 失败：红色渐变背景 + ❌ 图标
- 5秒后自动关闭
